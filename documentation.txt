                                        Module 0 : Environment Setup
                    ----------------------------------------------------------------------
1. npm init - y 
    initialize node project. package.json contains all the information about the project. every node project has one.

2. npm install -D typescript
    -D : --save-dev : this package is only needed while depeloping
    => nodes_modules : where npm stores every installed packages

3. npx tsc --init
    => typescript configuration
    => creates tsconfig.json : tells the typescript compiler how to convert .ts file into .js
    => why we need .js? because, browser and node.js execute .js files not typescript.

now we paste the following code in tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "CommonJS",
    "rootDir": "./src",
    "outDir": "./dist",

    "strict": true,

    "esModuleInterop": true,

    "skipLibCheck": true
  }
}
here,   target          : javascript version to generate
        module          : module system CommonJS works good with node.js
        rootDir         : src code lives here
        outDir          : where .js code will be written
        strict          : Enables strict type checking to catch bugs early
        esModuleInterop : makes importing CommonJS modules easiers
        skipLibCheck    : speed up compilation by skipping checks on library declaration files
    
4. npm install -D @types/node
    install node.js type definition. automatically typescript does not know about node.js api's like console, process, or fs

        to check who created the data.

==> private key signs the Transactions
        Transactions -> private key -> signature
==> public key?

        Transaction + signature + public key -> valid? -> true\false
so, 
        private key     ->      signature

=> now, do we signature the entire Transaction?
- no, we sign the hash, because the entire Transaction is large, and it will be slow to sign this. 
so we sign the hash (which is unique for each transaction), and does our job perfectly.

finally, 
        transaction -> SHA256 -> hash -> private key -> signature
